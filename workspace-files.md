## 双层 AGENTS.md 架构

每个 Agent 拥有两份配置文件，职责不同、优先级不同：

| 层级 | 路径 | 用途 | 优先级 |
|------|------|------|--------|
| **系统提示** | `agents/{id}/agent/AGENTS.md` | 角色定义 + 行为准则 + 协作模式 | 最高（Gateway 启动加载） |
| **工作空间** | `workspace-{id}/AGENTS.md` | 详细工作流 + Discord Bot ID + 工具备忘 | 运行时上下文 |

- **系统提示层**：Gateway 启动时加载到 session-level 缓存（无 TTL），修改后需 `openclaw gateway restart`
- **工作空间层**：运行时按需读取，修改即时生效
- **双模式协作**：日常模式（任务来自智多星）+ Ship Faster 模式（任务来自 Aria，可直接 @mention）

## 核心原则 (SOUL.md)

1. **产物优先** — 对话必须转化为文件产物
2. **有用而不是表演有用** — 拒绝寒暄直接提供选项
3. **一次只问一个问题** — 降低人类认知负载
4. **自愈与反思** — 任务失败必须总结入 Lore
