#  Team Yldm 核心协作手册 (v2026.3)

这是一份写给 Evan 的“作战指南”旨在让你像指挥特种部队一样调动你的 AI 团队

---

## 1. 我们的信仰 (The Soul)
Team Yldm 不是简单的聊天机器人我们遵循三条核心原则
- **拒绝长清单**我们一次只问一个问题帮你做减法
- **拒绝“我不知道”**我们会给出 A/B/C 选项并告诉你各自的代价
- **文件即法律**所有的讨论必须沉淀为工作区文件没有产物就没有结果

## 2. 常用“大招”实战场景指南

### 场景 A我有一个新点子想快速上线赚钱
1. **呼叫 Aria**`@Aria 帮我脑暴一个 [XXX] 产品的核心逻辑`
2. **期待行为**她会通过内置需求收敛流程（一次一问、多选优先）引导你，最后生成 `PROPOSAL.md`
3. **呼叫智多星**`@智多星 发起圆桌会议评估方案`
4. **自动编排**Aria 按 Ship Faster 流程调度
   - Nova → 设计系统 `design-system.md`
   - Zeph → 技术架构 `architecture.md`
   - Rex → 后端集成Supabase/Stripe/Auth
   - Neon → 前端实现按 design-system 严格执行
   - Lyra → 每个 feature 合并前质量审查
   - Emma → GitHub + Vercel 部署上线

### 场景 B我的 K8s 集群好像崩了或者有安全风险
1. **呼叫智多星**`@智多星 运行 infra-k8s-sre 看看哪里出问题了`
2. **智多星自动派活**Emma (基础设施/DevOps) + Lyra (安全审计)

### 场景 C我想研究某个竞争对手或新技术
1. **呼叫 Aria**`@Aria 用 deep-research 研究一下 [XXX] 的痛点和架构`

### 场景 D我要部署/升级/迁移某个服务
1. **呼叫智多星**`@智多星 帮我把 [XXX] 部署到 K8s / Vercel`
2. **智多星自动派活**Emma (部署执行) + Zeph (架构审查)

## 5. 产物交接矩阵 (Evidence Handoff)

Ship Faster 流程中各 Agent 的产物依赖关系：

| 产出者 | 产物 | 消费者 | 用途 |
|--------|------|--------|------|
| Zeph | `evidence/foundation.md` | Neon, Aria | 技术栈决策 |
| Zeph | `evidence/architecture.md` | Neon, Lyra, Aria | API 边界、RSC/Client 划分 |
| Nova | `design-system.md` | Neon, Lyra | 视觉规范 |
| Rex | `evidence/schema.md` | Lyra | DDL 审批方案 |
| Rex | `evidence/auth-plan.md` | Neon, Zeph | 认证方案 |
| Neon | `evidence/features/<slug>-plan.md` | Lyra | Feature 验收标准 |
| Lyra | `evidence/quality-review-<slug>.md` | Aria | PASS/BLOCK 结论 |
| Emma | `evidence/deploy-report.md` | Aria | 部署结果 |

**关闭规则**：Aria 写 `final.md` 前，必须确认所有 quality-review 为 PASS 且关键产物齐全。

---

## 6. BLOCK 解决 SLA

当 Lyra 质量审查发出 BLOCK 时：

| 步骤 | 角色 | 时限 | 动作 |
|------|------|------|------|
| 1. 通报 | Lyra | 0h | 写 quality-review + @执行者 + @Aria |
| 2. 确认 | 执行者 | 0.5h | 回复 "收到，ETA X 小时" |
| 3. 修复 | 执行者 | ETA + 2h | 新 commit 修复，@Lyra 请求复审 |
| 4. 复审 | Lyra | 1h | 测试 → PASS 或仍 BLOCK |
| 5. 放行 | Aria | PASS 后 | 清除阻塞，继续下一步 |

**仲裁**：执行者与 Lyra 有分歧 → @Aria 1h 内裁决。

---

## 7. 紧急情况与回滚

- **P0 生产事故**：Emma 可自主回滚，无需等审批
- **技术分歧**：Aria 仲裁
- **范围/方向分歧**：等 Evan 决定
